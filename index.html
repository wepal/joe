GitHub Tipps für Joe

Joes cooles Repository zum Ausprobieren von Git & GitHub

Willkommen, Joe! Hier findest du einige praktische Tipps, die dir den Einstieg und das goonen in GitHub erleichtern. Viel Spaß beim goonen auf benni! 🚀
1

Repository erstellen

Starte immer mit einem neuen Repository. Klicke auf „New repository" auf deiner GitHub-Startseite, gib einen Namen ein und initialisiere es optional mit einer README.md. So hat dein Projekt sofort eine Beschreibung und du kannst es direkt star wars the klonen wars.

2

Commits sinnvoll benennen oder auch nicht

Schreibe aussagekräftige Commit-Nachrichten, z. B. feat: Startseite hinzugefügt oder fix: Tippfehler in README behoben. Gute Nachrichten erleichtern es dir und anderen, den Verlauf des Projekts zu verstehen.

3

Branches nutzen

Entwickle neue Features oder Bugfixes immer in einem eigenen Branch, z. B. git checkout -b feature/mein-feature. So bleibt der main-Branch stabil, und du kannst Änderungen über einen Pull Request einreichen und reviewen lassen.

4

Pull Requests & Reviews

Öffne einen Pull Request (PR), wenn dein Branch fertig ist. Beschreibe, was du geändert hast, und lade Teammitglieder zum Review ein. Feedback im PR hilft dabei, Fehler frühzeitig zu entdecken, bevor der Code in main landet.

5

Issues verwenden

Nutze Issues, um Aufgaben, Bugs oder Ideen festzuhalten. Du kannst Issues mit Labels (z. B. bug, enhancement) kategorisieren und sie Teammitgliedern zuweisen. So behältst du immer den Überblick über offene Aufgaben.

6

.gitignore nicht vergessen

Füge eine .gitignore-Datei hinzu, damit temporäre Dateien, Passwörter oder Build-Artefakte (z. B. node_modules/, .env) nicht ins Repository gelangen. GitHub bietet fertige Vorlagen für viele Programmiersprachen an.

Tutorial: Git-Grundlagen in der Praxis

Schritt 1

Repository klonen

  1. Öffne ein Terminal (z. B. PowerShell, Git Bash oder das integrierte Terminal).
  2. Gehe auf GitHub zur gewünschten Repository-Seite und klicke auf den grünen „Code"-Button.
  3. Kopiere die HTTPS-URL (z. B. https://github.com/nutzer/repo.git).
  4. Führe im Terminal folgenden Befehl aus:
    # Repository in den aktuellen Ordner klonen git clone https://github.com/nutzer/repo.git # Danach in den neuen Ordner wechseln cd repo
Das Repository wird als neuer Unterordner angelegt. Git lädt automatisch alle Dateien und den gesamten Verlauf herunter.
  1. Öffne VSCode. Wenn noch kein Ordner geöffnet ist, siehst du auf der Startseite den Button „Clone Git Repository…" – klicke darauf.
  2. Alternativ: Drücke Strg+Shift+P und tippe „Git: Clone", dann Enter.
  3. Füge die Repository-URL ein (z. B. https://github.com/nutzer/repo.git) und drücke Enter.
  4. Wähle einen lokalen Ordner als Ziel aus.
  5. VSCode fragt, ob du das geklonte Repository direkt öffnen möchtest – klicke auf „Open".
Die URL findest du auf GitHub unter dem grünen „Code"-Button → HTTPS-Tab.
  1. Öffne das Repository auf github.com.
  2. Klicke auf den grünen „Code"-Button.
  3. Wähle den Tab „GitHub Desktop" und klicke auf „Open with GitHub Desktop" – das öffnet GitHub Desktop und klont das Repo automatisch.
  4. Wenn du lieber direkt im Browser arbeitest (ohne lokale Kopie), klicke stattdessen auf „Codespace", um eine cloudbasierte Entwicklungsumgebung zu starten.
Für reines Bearbeiten einzelner Dateien kannst du diese auch direkt im Browser öffnen und bearbeiten – ohne klonen.
Schritt 2

Branch erstellen

  1. Stelle sicher, dass du im Repository-Ordner bist und der main-Branch aktuell ist:
    git checkout main git pull
  2. Erstelle einen neuen Branch und wechsle direkt dazu:
    # Branch erstellen und sofort wechseln git checkout -b feature/mein-feature # Alternativ (moderner Weg): git switch -c feature/mein-feature
  3. Überprüfe, auf welchem Branch du bist:
    git branch
Benenne Branches aussagekräftig, z. B. feature/login-seite oder fix/tippfehler-header.
  1. Klicke unten links in der Statusleiste auf den aktuellen Branch-Namen (z. B. „main").
  2. Es öffnet sich eine Dropdown-Liste – klicke auf „+ Create new branch…".
  3. Gib den Namen des neuen Branches ein (z. B. feature/mein-feature) und drücke Enter.
  4. VSCode wechselt automatisch auf den neuen Branch – erkennbar unten links am aktualisierten Branch-Namen.
Alternativ: Strg+Shift+P → „Git: Create Branch" eintippen.
  1. Öffne dein Repository auf github.com.
  2. Klicke oben links auf das Branch-Dropdown (zeigt z. B. „main" an).
  3. Tippe den Namen des neuen Branches in das Suchfeld (z. B. feature/mein-feature).
  4. Es erscheint die Option „Create branch: feature/mein-feature from 'main'" – klicke darauf.
Branches, die direkt auf GitHub erstellt werden, existieren zunächst nur remote. Führe lokal git pull aus, um sie herunterzuladen.
Schritt 3

Änderungen committen

  1. Schaue dir an, was sich geändert hat:
    git status
  2. Füge die geänderten Dateien zur sogenannten „Staging Area" hinzu:
    # Eine einzelne Datei stagen git add index.html # Alle geänderten Dateien auf einmal stagen git add .
  3. Erstelle den Commit mit einer aussagekräftigen Nachricht:
    git commit -m "feat: Startseite mit Tutorial ergänzt"
  4. Lade den Branch auf GitHub hoch:
    # Beim ersten Mal mit -u, damit Git den Remote-Branch kennt git push -u origin feature/mein-feature # Danach reicht einfach: git push
  1. Klicke in der linken Sidebar auf das Source Control-Icon (das Verzweigungs-Symbol, oder Strg+Shift+G).
  2. Unter „Changes" siehst du alle geänderten Dateien. Klicke auf das „+"-Icon neben einer Datei, um sie zu stagen (oder auf das „+" oben, um alle zu stagen).
  3. Tippe oben im Textfeld eine Commit-Nachricht ein, z. B. feat: Startseite aktualisiert.
  4. Klicke auf den blauen „Commit"-Button.
  5. Klicke danach auf „Sync Changes" (oder „Publish Branch" beim ersten Mal), um den Commit auf GitHub zu pushen.
VSCode zeigt im Commit-Button auch an, wie viele Commits noch nicht gepusht wurden (z. B. „Commit ↑1").
  1. Navigiere im Browser zu der Datei, die du bearbeiten möchtest.
  2. Klicke oben rechts auf das Stift-Icon (Edit this file).
  3. Nimm deine Änderungen direkt im Browser vor.
  4. Scrolle nach unten zum Abschnitt „Commit changes".
  5. Gib eine kurze Beschreibung ein, wähle optional „Commit directly to the [branch]-branch" und klicke auf „Commit changes".
Diese Methode eignet sich nur für kleine Änderungen an einzelnen Dateien. Für größere Änderungen lieber CLI oder VSCode nutzen.
Schritt 4

Pull Request erstellen

  1. Stelle sicher, dass dein Branch gepusht ist (siehe Schritt 3).
  2. Installiere die GitHub CLI (gh) von cli.github.com und melde dich an:
    gh auth login
  3. Erstelle den Pull Request direkt aus dem Terminal:
    # Interaktiver Assistent führt dich durch den Prozess gh pr create # Oder direkt mit Titel und Beschreibung: gh pr create --title "feat: Tutorial hinzugefügt" --body "Beschreibung der Änderungen"
  4. Du erhältst einen Link zum PR auf GitHub, den du im Browser öffnen kannst.
gh pr list zeigt alle offenen PRs, gh pr merge merged einen PR direkt aus dem Terminal.
  1. Installiere die VSCode-Erweiterung „GitHub Pull Requests" (von GitHub).
  2. Nach dem Pushen deines Branches erscheint in der Source-Control-Ansicht ein Button „Create Pull Request" – klicke darauf.
  3. Es öffnet sich ein Panel, in dem du Titel, Beschreibung, Reviewer und Ziel-Branch festlegen kannst.
  4. Klicke auf „Create". Der PR wird direkt auf GitHub erstellt.
  5. Du kannst PRs auch direkt in VSCode reviewen: Unter dem GitHub-Icon in der Sidebar findest du alle offenen PRs.
Die Erweiterung erlaubt es dir, Kommentare zu lesen und zu schreiben, ohne den Browser zu öffnen.
  1. Gehe auf github.com zu deinem Repository. Nach einem frischen Push erscheint oben ein gelbes Banner „Compare & pull request" – klicke darauf.
  2. Alternativ: Klicke auf den Tab „Pull requests" → „New pull request".
  3. Wähle als base den Ziel-Branch (meistens main) und als compare deinen Feature-Branch.
  4. Füge einen Titel und eine Beschreibung hinzu, weise optional Reviewer zu.
  5. Klicke auf „Create pull request". Sobald alle Reviewer zugestimmt haben, kannst du auf „Merge pull request" klicken.
Nach dem Merge kannst du den Feature-Branch mit „Delete branch" direkt auf GitHub löschen, um das Repository aufgeräumt zu halten.
